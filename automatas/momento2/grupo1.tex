\begin{abstract}
En este proyecto se desarrolló una estación de confort para mejorar las condiciones 
de un puesto de estudio o trabajo. El sistema permite medir la temperatura y el nivel de 
iluminación del entorno, mostrando los datos en una pantalla LCD I2C. Además, el sistema 
toma decisiones automáticas como activar ventilación mediante un motor DC y regular la 
iluminación utilizando PWM. Se implementaron modos de operación manual y automático, 
junto con técnicas como antirrebote y temporización con millis().
\end{abstract}

\begin{IEEEkeywords}
Digital, antirrebote, estados, analógico, PWM, DS18B20, actuador con transistor, LCD I2C, millis()
\end{IEEEkeywords}

\section{Introducción}
Actualmente, las condiciones ambientales influyen directamente en el rendimiento académico y laboral. 
Factores como la temperatura y la iluminación pueden afectar la concentración y la productividad. 
Por ello, en este proyecto se propone el diseño de un sistema automatizado que supervise y mejore dichas 
condiciones en un puesto de estudio.

\section{Objetivos}
\subsection{Objetivo General}
Diseñar una estación de confort que supervise y mejore las condiciones de temperatura e iluminación 
en un puesto de estudio, utilizando sensores y actuadores controlados por un microcontrolador.
\subsection{Objetivos Específicos}
\begin{itemize}
    \item Medir la temperatura ambiente utilizando un sensor DS18B20.
    \item Medir el nivel de iluminación con un sensor LDR.
    \item Mostrar los datos en una pantalla LCD I2C.
    \item Implementar un sistema de control automático y manual del sistema.
    \item Controlar ventilación mediante un motor DC.
    \item Regular la iluminación utilizando PWM.
\end{itemize}

\section{Metodología}
Para el desarrollo del proyecto se trabajó en equipo dividiendo tareas como programación, 
montaje del circuito y documentación. Se utilizó una placa Arduino como microcontrolador principal, 
junto con sensores y actuadores.
Primero se realizó el diseño del circuito en protoboard, conectando el sensor de temperatura DS18B20, 
la LDR para medición de luz, la pantalla LCD I2C, un motor DC y un LED para simular la iluminación.
Luego se desarrolló el código en Arduino IDE, implementando lectura de sensores, control 
de actuadores y visualización de datos. Se utilizó la función millis() para evitar la congelación del 
sistema con delay() y mejorar la fluidez.

\section{Diseño del Sistema}
\subsection{Diagrama de Funcionamiento}
El sistema funciona a partir de la lectura constante de los sensores. Dependiendo de los valores 
obtenidos, el sistema toma decisiones automáticamente o permite control manual.
\subsection{Condiciones de Operación}

\section{Desarrollo}
\subsection{Lectura de sensores}
El sensor DS18B20 se utilizó para medir la temperatura con buena precisión. Este sensor trabaja
de forma digital, lo que facilita su lectura. Por otro lado la LDR entrega valores analógicos
que dependen de la cantidad de luz recibida.
\subsection{Visualización en el LCD}
Se utilizó una pantalla LCD con comunicación I2C para mostrar:
\begin{itemize}
    \item Temperatura actual en °C.
    \item Nivel de iluminación.
    \item Modo de operación (Automático o Manual).
\end{itemize}
Esto permite al usuario entender fácilmente el estado del sistema.

\section{Decisiones técnicas}
\begin{itemize}
    \item Se usó una resistencia de 220 ohmios para el led, así aprovechamos todo su brillo sin quemarlo.
    \item Se usó una resistencia de 4.7 kOhms para el sensor ya que esta evita lecturas incorrectas del 
sensor hacia el Arduino.
    \item Usamos millis() y evitamos el uso de delay() para el antirrebote y que el sistema pudiera
seguir trabajando en los sensores y que no se detuviera mientras verifica las pulsaciones.
    \item Los botones se configuraron como INPUT\_PULLUP para evitar el uso de resistencias externas en el montaje.
    \item Usamos un LCD I2C para aprovechar una conexión con menos cables y así tener más espacio para otros componentes.
\end{itemize}
